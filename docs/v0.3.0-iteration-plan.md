# v0.3.0 迭代计划

> 目标日期：2026-03-14
> 状态：已实施 (2026-03-13)

## 问题诊断

### 核心 Bug：skill 调用只生成 JSON，无法渲染图

**现象**：用户调用 excalidraw-agent-local-kit skill 后，AI 只输出一个 `.elements.json` 文件，用户无法直接在 Excalidraw 中打开或看到可视化图。

**根因**：

1. `SKILL.md` 的 `allowed-tools` 只有 `Read, Write, Bash, Grep, Glob`，**不包含任何 MCP 工具**
2. Skill 被触发后，AI 无权调用 `mcp__excalidraw__create_view` 渲染图，也无权调用 `mcp__excalidraw__export_to_excalidraw` 导出
3. AI 只能用 Write 写文件 → 用户拿到的只是原始 JSON

### 文件格式断层

| 格式 | 用途 | 能否直接导入 Excalidraw |
|------|------|------------------------|
| `*.elements.json` | 本地 MCP truth source | ❌ 不能 |
| `*.excalidraw` | 原生场景文件 | ✅ 可以 |

Skill 只产出前者，没有自动转换到后者。

## 改动清单

### P0：allowed-tools 加入本地 MCP

**文件**：`SKILL.md`

```yaml
# 当前
allowed-tools: Read, Write, Bash, Grep, Glob

# 改为
allowed-tools: Read, Write, Bash, Grep, Glob, mcp__excalidraw__read_me, mcp__excalidraw__create_view, mcp__excalidraw__export_to_excalidraw, mcp__excalidraw__save_checkpoint, mcp__excalidraw__read_checkpoint
```

**效果**：skill 执行时可以直接调用本地 MCP 渲染和导出。

### P0：更新 Suggested Workflow（已实施）

**文件**：`SKILL.md`

实际实施结果 — 简化为两个模式（移除 Browser-import，合并进 Local-first）：

```markdown
### Local-first mode（默认）

1. 读取已有 `*.elements.json`（如存在），patch 而非重建
2. 生成或 patch 元素数组，严格遵守 Layout Defaults 间距规范
3. 保存 `*.elements.json` 作为 truth source
4. 运行 `node scripts/export_native_scene.js` 生成 `.excalidraw` 文件
5. 告知用户文件路径，可在 excalidraw.com 直接打开
6. （可选）如客户端支持 GUI 渲染，调用 `create_view` 预览

### Online-share mode

1. 完成 local-first mode 步骤 1-4
2. 调用 `mcp export_to_excalidraw` 获取链接
3. 验证分享页面文本可见
```

**改动说明**：`create_view` 在 CLI 终端无 GUI 渲染能力，降级为可选。默认路径改为自动导出 `.excalidraw` 文件。Browser-import 与 Local-first 重复，已合并。

### P1：MCP fallback 策略

**文件**：`SKILL.md`

新增 fallback 段落：

- 如果本地 MCP (`mcp__excalidraw__`) 不可用，尝试远程 MCP (`mcp__excalidraw_remote__`)
- 如果都不可用，走 `Bash` 调用 `export_native_scene.js` 生成 `.excalidraw` 文件
- 最终兜底：直接写 native `.excalidraw` JSON（手动构造完整 schema）

### P2：自动检测输出意图（已实施）

**文件**：`SKILL.md`

实际实施结果：

- 用户说"画个图" / "帮我做个流程图" / "导出" → Local-first mode
- 用户说"分享链接" / "发给别人" → Online-share mode
- 默认 → Local-first mode

## 不改的部分

- `scripts/export_native_scene.js` 和 `export_native_scene_impl.mjs` — 逻辑正确，无需修改
- `examples/` 模板 — 格式正确
- 远程 MCP 工具 — 不加入 allowed-tools（按需手动调用即可，避免 skill 默认走远程）

## 验证标准

1. 调用 skill 说"画一个简单的流程图" → 本地 MCP 弹出可视化窗口 ✅
2. 调用 skill 说"导出成 excalidraw 文件" → 生成 `.excalidraw` 文件，可在 excalidraw.com 打开 ✅
3. 调用 skill 说"给我分享链接" → 返回 excalidraw.com 链接，打开后文本可见 ✅
4. 本地 MCP 不可用时 → fallback 到 export script，用户拿到 `.excalidraw` 文件 ✅

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 本地 MCP 未启动 | create_view 失败 | P1 fallback 策略兜底 |
| allowed-tools 过多导致 skill token 开销增大 | 略微增加 prompt 长度 | 只加必要的 5 个本地 MCP 工具 |
| Codex 不支持 MCP 调用 | Codex 执行 skill 时仍然只出 JSON | Codex 路径走 Bash + export script |
